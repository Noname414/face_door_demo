## 完整接線圖 (Servo 版)

```
┌─────────────────────────────────────────────────────────────────┐
│                         Arduino Uno / Nano                      │
│                                                                 │
│        5V ────┬────┬──── (電源)                                 │
│               │    │                                            │
│       GND ────┼────┼──── (接地)                                 │
│               │    │                                            │
│        D5 ────┼────┼──── (藍牙 RX，接收資料)                    │
│        D6 ────┼────┼──── (藍牙 TX，傳送資料)                    │
│        D7 ────┼────┼──── (藍牙 STATE，可選)                     │
│        D9 ────┼────┼──── (伺服訊號，PWM)                        │
│               │    │                                            │
└───────────────┼────┼────────────────────────────────────────────┘
                │    │
        ┌───────┘    └────────┐
        │                     │
        ▼                     ▼
┌──────────────┐      ┌──────────────┐
│  HC-05/HC-06 │      │  SG90 Servo  │
│  藍牙模組     │      │  伺服馬達     │
├──────────────┤      ├──────────────┤
│ VCC → 5V     │      │ 紅線 → 5V    │
│ GND → GND    │      │ 棕線 → GND   │
│ TXD → D5     │      │ 橘線 → D9    │
│ RXD → D6     │      └──────────────┘
│ STATE → D7   │
│ EN → 不接    │
└──────────────┘
```

### 方案 A：轉接板版藍牙模組 (推薦) ✅

大多數市售模組都是轉接板版本，PCB 上有穩壓晶片和分壓電路。

**實體接線步驟：**

1. **斷電操作** — 先不要插 USB 或電源
2. 藍牙模組 VCC → Arduino **5V**
3. 藍牙模組 GND → Arduino **GND**
4. 藍牙模組 TXD → Arduino **D5** ✅
5. 藍牙模組 RXD → Arduino **D6** ✅ (板載分壓可直連)
6. 藍牙模組 EN 不接（預設資料模式）
7. 藍牙模組 STATE → Arduino **D7** (可選，不接也行)
8. 伺服馬達紅線 → Arduino **5V**
9. 伺服馬達棕/黑線 → Arduino **GND**
10. 伺服馬達橘/黃線 → Arduino **D9** (PWM 訊號)
11. 上傳程式後再插 USB 供電

### 詳細接線對照表

| 元件 | 腳位 | 顏色參考 | → | Arduino |
|------|------|---------|---|---------|
| **HC-05 轉接板** | VCC | 紅 | → | **5V** |
| | GND | 黑 | → | **GND** |
| | TXD | 綠/白 | → | **D5** (RX) |
| | RXD | 藍/白 | → | **D6** (TX) |
| | EN/KEY | - | → | 不接 |

| | Signal (橘/黃) | 橘 | → | **D9** |

### 方案 B：裸藍牙模組 (6 針腳版本)

如果你買的是沒有轉接板的裸晶片模組，需要額外分壓電路。

```
                               ┌─────────────────┐
                               │   Arduino Uno   │
┌─────────────────┐            ├─────────────────┤
│  HC-05 裸模組    │            │                 │
├─────────────────┤            │                 │
│ EN              │            │                 │ (不接)
│ VCC      ───────┼────────────┤ 3.3V            │ ⚠️ 不是 5V！
│ GND      ───────┼────────────┤ GND             │
│ TXD      ───────┼────────────┤ D5              │
│                 │   1kΩ      │                 │
│ RXD      ───────┼───[1kΩ]────┤ D6              │
│                 │      │      │                 │
│                 │     2kΩ     │                 │
│                 │      └──────┤ GND             │
│ STATE           │            │                 │ (不接)
└─────────────────┘            └─────────────────┘

伺服接法同上 (紅→5V, 棕→GND, 橘→D9)
```

**分壓電阻接法：**
```
Arduino D6 ──[1kΩ電阻]── 藍牙RXD
                   │
                 [2kΩ電阻]
                   │
                  GND
```
這會將 Arduino 的 5V 訊號降到 3.3V 保護藍牙模組。

### 角度設定 (face_door.ino)

```cpp
const int SERVO_CLOSED_ANGLE = 0;    // 關門角度
const int SERVO_OPEN_ANGLE   = 90;   // 開門角度
```

**想反過來？** 只要對調兩個數字：
```cpp
const int SERVO_CLOSED_ANGLE = 90;   // 改成這樣
const int SERVO_OPEN_ANGLE   = 0;
```

**想要更大開門角度？** 最大 180°：
```cpp
const int SERVO_OPEN_ANGLE   = 180;  // 完全轉到底
```

## 檢查清藍牙 VCC 接對 (轉接板→5V，裸模組→3.3V)
- [ ] TXD/RXD 交叉接對 (模組TXD→D5，模組RXD→D6)
- [ ] 伺服供電正確 (紅→5V, 棕→GND, 橘→D9)
- [ ] 伺服轉臂 (舵盤) 已安裝好

**上傳時：**
- [ ] 先拔除 D5/D6 的藍牙連線 (避免與上傳衝突)
- [ ] 上傳完成後再插回

**測試步驟：**
1. 上傳程式，打開序列埠監控視窗 (9600 baud)
2. 應該看到 `[BOOT] Face Door (Servo) ready.`
3. 伺服應該會移動到 0° (關門位置)
4. 手機連接藍牙 (PIN: 1234 或 0000)
5. 用藍牙序列埠工具發送 `OPEN\n`
6. 應該看到 `[DOOR] OPENED.` 且伺服轉到 90°
7. 3 秒後自動回到 0° (關門)

**常見藍牙序列埠 App：**
- Android: **Serial Bluetooth Terminal** (推薦)
- iOS: BLE Terminal (需 BLE 模組，HC-05/HC-06 不支援)
- 電腦: PuTTY、RealTerm、Arduino IDE Serial Monitor

## 疑難排解

| 伺服不動 | 訊號線沒插 / 沒電 | 確認 橘線→D9，紅線→5V 都接好 |
| 伺服抖動 / 亂轉 | 電源不穩 / 干擾 | 用外部 5V 電源 + 加去耦電容 |
| 序列埠監控亂碼 | baud rate 不符 | 確認監控視窗設為 **9600** (不是 115200) |
| Arduino 不斷重啟 | 電流過載 | SG90 通常沒問題，大型伺服用外部電源 |
| Python 找不到 COM 埠 | 藍牙未配對 / 埠號錯 | 檢查裝置管理員的「標準序列埠 (傳出)」或 1234 |
| 配對成功但無法通訊 | RX/TX 接反 | 確認 模組TXD→D5，模組RXD→D6 |
| 上傳程式失敗 | D0/D1 被佔用 | 上傳時拔掉 D5/D6 藍牙線 |
| 繼電器不動作 | 訊號反向 | 改用 LOW 觸發 (見上方說明) |
| 序列埠監控亂碼 | baud rate 不符 | 確認監控視窗設為 115200 |
| Arduino 不斷重啟 | 電流過載 | 電磁鎖用獨立 12V 電源，不可接 Arduino |

## 進階：電源供應建議

### 方案 1 (測試/展示用) — 單 SG90 ✅

```
USB 5V → Arduino → 藍牙 + SG90 伺服
```

SG90 空載電流約 100mA，堵轉約 500mA，USB 供電夠用。

### 方案 2 (多顆伺服 / 大型伺服)

```
外部 5V 2A 電源 ───┬→ Arduino Vin (或 5V pin)
                  ├→ 伺服 VCC (紅線)
                  └→ 藍牙 VCC
                  
⚠️ 所有 GND 必須共接！
```

**推薦電源：**
- 5V 2A USB 充電器 + Type-C/Micro-USB 轉接線
- 5V 穩壓模組 (LM2596) 從 12V 降壓
